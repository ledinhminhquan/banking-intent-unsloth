\section{Introduction}
% ==============================================================================

This report documents the implementation and evaluation of a banking intent
classification system built for Project 2 of the CSC15012 course. The task is
to fine-tune a large language model (LLM) using \textbf{Unsloth} and
\textbf{QLoRA} (Quantized Low-Rank Adaptation) on the \textbf{BANKING77}
dataset so that the model can accurately predict the intent behind a customer's
banking-related message.

The BANKING77 dataset, introduced by Casanueva et al.~\cite{casanueva2020},
contains 13,083 annotated customer service queries across 77 fine-grained
intent categories that cover common banking operations such as card activation,
transfers, balance inquiries, and more.

The main objectives of this project are:
\begin{itemize}[noitemsep]
    \item Sample and construct a suitable subset of the BANKING77 dataset.
    \item Preprocess data and split into training and testing sets.
    \item Fine-tune a text classification model using Unsloth with LoRA.
    \item Evaluate and compare the fine-tuned model on an independent test set.
    \item Implement a standalone inference class for deployment.
\end{itemize}

\section{Data Preparation and Processing}
\label{sec:data}
% ==============================================================================

\subsection{Dataset Overview}

The BANKING77 dataset is available on HuggingFace at
\texttt{PolyAI/banking77}. It consists of:
\begin{itemize}[noitemsep]
    \item \textbf{Training set:} 10,003 samples
    \item \textbf{Test set:} 3,080 samples
    \item \textbf{Number of intents:} 77 categories
\end{itemize}

Example intent labels include \texttt{activate\_my\_card},
\texttt{card\_delivery\_estimate}, \texttt{change\_pin},
\texttt{cancel\_transfer}, \texttt{lost\_or\_stolen\_card}, etc.

\subsection{Sampling Strategy}

To ensure training can be completed efficiently with available computational
resources (Google Colab Pro with NVIDIA H100 GPU), we sampled a \textbf{subset}
of the full dataset:

\begin{itemize}[noitemsep]
    \item \textbf{Training:} 40 samples per intent class
          $\rightarrow$ $40 \times 77 = 3{,}080$ samples
    \item \textbf{Test:} 10 samples per intent class
          $\rightarrow$ $10 \times 77 = 770$ samples
\end{itemize}

Sampling was performed with a fixed random seed (\texttt{seed=42}) for
reproducibility.

\subsection{Preprocessing Steps}

\begin{enumerate}[noitemsep]
    \item \textbf{Text normalization:} All text was converted to lowercase and
          extra whitespace was removed.
    \item \textbf{Label mapping:} A bidirectional mapping
          (\texttt{id2label} / \texttt{label2id}) was created and saved to
          \texttt{sample\_data/label\_map.json}.
    \item \textbf{Format conversion:} Data was saved in CSV format with columns
          \texttt{text}, \texttt{label} (integer ID), and
          \texttt{label\_text} (human-readable intent name).
    \item \textbf{Shuffling:} Both train and test sets were shuffled.
\end{enumerate}

\section{Fine-Tuning with Unsloth}
\label{sec:finetuning}
% ==============================================================================

\subsection{Model Selection}

We used the \textbf{LLaMA 3.2 3B Instruct} model loaded in 4-bit quantization
via Unsloth:

\begin{center}
    \texttt{unsloth/Llama-3.2-3B-Instruct-bnb-4bit}
\end{center}

This model provides a good balance between performance and resource
requirements. The 4-bit quantization reduces memory usage by $\sim$4$\times$
compared to full precision, enabling fine-tuning on a single GPU.

\subsection{LoRA Configuration}

Low-Rank Adaptation (LoRA)~\cite{hu2021lora} was applied to the following
attention and feed-forward projection layers:

\begin{table}[H]
    \centering
    \caption{LoRA hyperparameters.}
    \label{tab:lora}
    \begin{tabular}{ll}
        \toprule
        \textbf{Parameter}     & \textbf{Value}                                        \\
        \midrule
        LoRA rank ($r$)        & 16                                                    \\
        LoRA alpha ($\alpha$)  & 16                                                    \\
        LoRA dropout           & 0.0                                                   \\
        Target modules         & \texttt{q\_proj, k\_proj, v\_proj, o\_proj}            \\
                               & \texttt{gate\_proj, up\_proj, down\_proj}              \\
        Bias                   & none                                                  \\
        Gradient checkpointing & Unsloth (memory-efficient)                            \\
        \bottomrule
    \end{tabular}
\end{table}

\subsection{Training Format}

We used \textbf{Supervised Fine-Tuning (SFT)} with an Alpaca-style prompt
template. Each training example was formatted as:

\begin{lstlisting}[language={},caption={Prompt template for SFT.}]
Below is an instruction that describes a task, paired with an input
that provides further context. Write a response that appropriately
completes the request.

### Instruction:
Classify the intent of the following banking customer message.
Reply with only the intent label, nothing else.

### Input:
{customer_message}

### Response:
{intent_label}<|end_of_text|>
\end{lstlisting}

During inference, the ``\#\#\# Response:'' section is left empty for the model
to generate.

\section{Inference Implementation}
\label{sec:inference}
% ==============================================================================

The inference module (\texttt{scripts/inference.py}) implements the required
\texttt{IntentClassification} class with two main methods:

\begin{lstlisting}[language=Python, caption={Inference class interface.}]
class IntentClassification:
    def __init__(self, model_path):
        """
        Load config, tokenizer, model checkpoint,
        and label mapping from model_path (YAML config).
        """
        ...

    def __call__(self, message):
        """
        Receive a banking customer message string
        and return the predicted intent label.
        """
        ...
        return predicted_label
\end{lstlisting}

\textbf{Key design decisions:}
\begin{itemize}[noitemsep]
    \item The \texttt{model\_path} argument points to a YAML configuration file
          (\texttt{configs/inference.yaml}) that specifies the checkpoint path,
          label map file, and generation settings.
    \item The model is loaded in 4-bit quantization mode for memory efficiency.
    \item Fuzzy label matching (\texttt{difflib.get\_close\_matches}) is used as
          a fallback when the generated text does not exactly match a known
          intent label.
    \item The inference script supports three modes: single message, batch
          evaluation, and interactive mode.
\end{itemize}

\textbf{Usage example:}
\begin{lstlisting}[language=Python, caption={Using the IntentClassification class.}]
from scripts.inference import IntentClassification

classifier = IntentClassification("configs/inference.yaml")
label = classifier("I need to change my PIN")
print(label)  # "change_pin"
\end{lstlisting}

\begin{itemize}[noitemsep]
    \item \textbf{scripts/preprocess\_data.py}: Downloads BANKING77 from
          HuggingFace, samples a subset, cleans text, and saves CSV files.
    \item \textbf{scripts/train.py}: Loads model with Unsloth, applies LoRA,
          trains with SFTTrainer, saves checkpoint, and evaluates on test set.
    \item \textbf{scripts/inference.py}: Standalone inference class that loads
          the saved checkpoint and predicts intent labels.
    \item \textbf{configs/}: YAML configuration files for all hyperparameters.
    \item \textbf{train.sh / inference.sh}: Shell scripts for one-click
          execution.
\end{itemize}

\section{Discussion and Conclusion}
\label{sec:conclusion}
% ==============================================================================

\subsection{Summary}

In this project, we successfully fine-tuned a \textbf{LLaMA 3.2 3B} model
using \textbf{Unsloth} and \textbf{QLoRA} for the task of banking intent
classification on the BANKING77 dataset. The key findings are:

\begin{itemize}[noitemsep]
    \item 4-bit quantization with LoRA enables efficient fine-tuning of a
          3-billion-parameter model on a single GPU.
    \item The Alpaca-style SFT approach can effectively reformulate
          classification as a text generation task.
    \item The model achieves competitive accuracy on the 77-class intent
          classification task even with a sampled subset of the data.
\end{itemize}

\subsection{Challenges}

\begin{itemize}[noitemsep]
    \item \textbf{Label matching:} Since the model generates free-form text,
          exact label matching is not always guaranteed. Fuzzy matching mitigates
          this but introduces a small error margin.
    \item \textbf{Inference speed:} Auto-regressive generation for
          classification is slower than a dedicated classification head. For
          production use, distilling into a smaller encoder model would be
          preferable.
\end{itemize}

\subsection{Future Work}

\begin{itemize}[noitemsep]
    \item Experiment with \texttt{FastLanguageModel.for\_sequence\_classification()}
          for direct classification head fine-tuning.
    \item Try larger models (e.g., LLaMA 3.1 8B) or alternative architectures.
    \item Use the full BANKING77 dataset instead of a sampled subset.
    \item Apply data augmentation techniques (paraphrasing, back-translation).
\end{itemize}
